     Aidele — Didaktische AI-Persona

     Context

     Young academics need methodological guidance when doing Situational
     Analysis for the first time. The existing analytical AI (map agent)
     operates within project data via naming acts. Aidele operates at the
     meta-level of method instruction — it observes the researcher's progress
     and teaches methodology through conversation, without writing to project
     data. The manual v2.0 (docs/manual.md) is ready as system prompt basis.

     Architecture

     ┌─────────────────┐    ┌──────────────────────┐    ┌─────────────────┐
     │  AidelePanel     │───▶│ POST /api/projects/  │───▶│ chat() (no tools)│
     │  (floating chat) │    │   [id]/aidele         │    │ client.ts        │
     │  sessionStorage  │    │                      │    └─────────────────┘
     │  for history     │    │ buildAideleContext()  │
     └─────────────────┘    │ logInteraction()      │
                            └──────────────────────┘

     Aidele is read-only — no tools, no naming acts, no DB writes except
     interaction logging.

     New Files (5)

     1. src/lib/server/ai/aidele-prompt.ts — System Prompt

     Condensed from docs/manual.md sections 1–6 (~4k tokens). Structure:

     1. Identity: "You are Aidele (AI + Aide + Adele E. Clarke) — a didactic
     companion..."
     2. What you do / don't do: teach methodology, observe progress, suggest
     next methodological moves. Do NOT produce analytical content, suggest
     elements, write memos, or evaluate quality
     3. Transactional ontology (Dewey/Bentley): naming as event, three
     observation levels
     4. CCS gradient: cue → characterization → specification, bidirectional,
     append-only
     5. Clarke's SA + critique: situation as unit, 12 heuristics, inter-actional
      limitation, material-mediality
     6. The unmarked: Spencer-Brown, Rancière, two observation levels
     7. Provenance: CCS vs grounding, orthogonal
     8. Data model summary: namings, participations, appearances, naming acts
     9. Three-layer hierarchy: data structure → list → canvas
     10. Map types: situational (flat ontology, relational analysis), SW/A
     (formations, universes of discourse), positional (disarticulation, axes,
     empty quadrants)
     11. Behavioral rules: non-evaluative, process-oriented, match researcher's
     language, reference "Clarke would suggest..." not "you should..."

     Export: AIDELE_SYSTEM_PROMPT constant.

     2. src/lib/server/ai/aidele-context.ts — Context Builder

     export async function buildAideleContext(
       projectId: string,
       currentPage: string,
       currentMapId?: string
     ): Promise<string>

     Queries DB and returns structured text:
     - Project name + description
     - Counts: documents, namings, maps, memos
     - Maps with types, element counts, designation profiles (top 5 by element
     count)
     - If currentMapId: lighter version of map context (elements with
     designations, relations summary, recent memos — no AI flags, no discussion
     summaries)
     - Designation profile aggregated project-wide
     - Current page indicator

     Reuses query patterns from +layout.server.ts (counts) and
     agent.ts:buildMapContext (map detail, lighter version).

     3. src/routes/api/projects/[projectId]/aidele/+server.ts — API Endpoint

     POST handler:
     - Input: { message, history: Array<{role, content}>, currentPage, 
     currentMapId? }
     - Builds context via buildAideleContext()
     - Calls chat({ system: AIDELE_SYSTEM_PROMPT, messages: [...history, {role: 
     'user', content: context + '\n\n' + message}], maxTokens: 1500 }) — no 
     tools
     - Logs via logInteraction(projectId, 'aidele', ...)
     - Returns: { response: string }

     History capped at last 40 messages to prevent unbounded token growth.

     4. src/lib/aidele/aideleState.svelte.ts — Frontend State

     Svelte 5 runes + sessionStorage per project:
     - messages: Array<{role: 'user'|'assistant', content: string}>
     - isOpen: boolean
     - loading: boolean
     - error: string | null
     - send(message, currentPage, currentMapId?) — POST to endpoint, append
     response
     - clear() — reset conversation

     Follow mapState.svelte.ts pattern: setAideleState() / getAideleState() via
     setContext/getContext.

     5. src/lib/aidele/AidelePanel.svelte — Chat UI

     Floating panel, bottom-right corner (position: fixed; bottom: 1rem; right: 
     1rem; z-index: 900):
     - Collapsed: toggle button with "Aidele" label
     - Expanded (~350×450px):
       - Header: "Aidele" + minimize + clear buttons
       - Scrollable message area (auto-scroll on new messages)
       - User messages right-aligned, Aidele messages left-aligned
       - Text input + send button at bottom
       - Loading indicator during AI response
     - Reads $page.url.pathname and $page.params for context detection
     - Dark theme matching app: #13151e background, #2a2d3a border, #a5b4fc
     accent

     Implementation Order

     1. aidele-prompt.ts (standalone, no dependencies)
     2. aidele-context.ts (depends on DB queries that already exist)
     3. aidele/+server.ts (depends on 1 + 2 + chat() + logInteraction())
     4. aideleState.svelte.ts (frontend, independent of backend)
     5. AidelePanel.svelte (depends on 4)
     6. Modify +layout.svelte (depends on 4 + 5)

     Steps 1–3 (backend) and step 4 (frontend state) can be done in parallel.

     Explicitly Deferred

     - Streaming responses (SSE) — simple POST/response for v1
     - Conversation persistence in DB — sessionStorage is sufficient
     - Editable system prompt (the persona toggle feature from
     project_ai_persona_toggle.md)
     - Markdown rendering in chat — plain text for v1
     - Suggested quick-start questions / prompt templates

     Verification

     1. Open a project → Aidele toggle visible in sidebar
     2. Click toggle → floating chat panel opens
     3. Send "Was ist Situational Analysis?" → get methodological explanation
     grounded in Clarke/Dewey-Bentley
     4. Navigate to a map → send "Wo stehe ich gerade?" → Aidele responds with
     context-aware assessment (map state, designation profile, methodological
     suggestions)
     5. Navigate to documents → context updates to documents page
     6. Check ai_interactions table: SELECT * FROM ai_interactions WHERE 
     request_type = 'aidele' → logged with provider, model, tokens
     7. Reload page → conversation preserved in sessionStorage
     8. Open new tab → conversation starts fresh (expected)
