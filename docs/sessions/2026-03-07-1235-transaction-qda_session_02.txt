  Was IST eine Situational Map als Datenobjekt?                                                                                                             
                                                                                                                                                            
  Eine Map ist ein Naming, das als perspective von sich selbst erscheint. Alles andere folgt daraus.                                                        
                                                                                                                                                            
  Messy Phase — aus der Datenstruktur gedacht:                                                                                                              
  - Namings werden erzeugt und erhalten eine Appearance auf der Map-als-Perspektive                                                                         
  - Jedes Element ist auf Designation-Stufe cue — registriert, aber nicht bestimmt                                                                          
  - Keine Participations, keine Relationen — nur eine Menge von Namings, die in diesem Perspektivraum erscheinen
  - Kein 2D-Zwang: die Menge ist ungeordnet, nicht räumlich                                                                                                 
                                                                                                                                                            
  Ordering Phase — was passiert hier strukturell?                                                                                                           
  - Participations werden erzeugt: "diese Namings ko-konstituieren sich"                                                                                    
  - Participations sind selbst Namings → sie können eigene Appearances haben (mode=relation, mit valence)                                                   
  - Designation bewegt sich: von cue zu characterization — das Naming wird bestimmter                                                                       
  - Phasen-Bildung: Cluster entstehen. Ein Cluster ist selbst ein Naming, das als Sub-Perspektive fungiert. Namings erscheinen aus diesem Cluster auf       
  bestimmte Weise — das ist kein Container, sondern eine neue Perspektive, die andere Appearances erzeugt                                                   
                                                                                                                                                            
  Clarkes Kategorien (human, nonhuman, discursive, etc.) als Phases:                                                                                        
  - Nicht vorgegeben, sondern emergent — sie entstehen als Namings, die als perspective erscheinen                                                          
  - In-vivo: der Forscher erkennt, dass bestimmte Namings zusammengehören → erzeugt ein Phase-Naming                                                        
  - Axial: der Forscher erkennt, dass Phases miteinander relatiert sind → Participations zwischen Phase-Namings                                             
                                                                                                                                                            
  Die Konsequenz: "Messy" und "Ordered" sind keine Modi, sondern ein Designation-Gradient.                                                                  
                                                                                                                                                            
  Es gibt keinen Umschalter. Die Map ist immer dieselbe Perspektive. Was sich ändert, ist der Bestimmtheitsgrad: Die naming_designations-Ketten der Elemente
   bewegen sich von cue Richtung characterization Richtung specification (und zurück). Die Frage "ist diese Map messy oder ordered?" ist beantwortet durch  
  den aggregierten Designation-Stand ihrer Elemente.

  Zur Richtung/Symmetrie:                                                                                                                                   
   
  Im Datenmodell ist das sauber getrennt:                                                                                                                   
  - Die Participation ist undirected — reines Ko-Konstitutionsverhältnis
  - Die Appearance (mode='relation') kann Richtung haben (directed_from/to) — aber das ist eine perspektivische Lesart, kein intrinsisches Merkmal          
                                                                                                                                                  
  Das heißt: Dieselbe Participation könnte auf Map A als A→B ("ermöglicht") erscheinen und auf Map B als B→A ("wird ermöglicht durch") oder als A↔B         
  (symmetrisch). Die Richtung gehört der Perspektive, nicht der Relation.                                                                                   
                                                                                                                                                            
  Das aktuelle UI macht das nicht sichtbar — es setzt implizit eine Richtung (wer zuerst geklickt wird = source), ohne zu fragen, ob die Relation überhaupt 
  gerichtet ist.                                            
                                                                                                                                                            
  Zur Inscription:                                          

  Die Participation ist selbst ein Naming — sie braucht eine eigene Inscription. Aber was? Das ist tatsächlich unklar. Aktuell fragt das UI separat nach    
  "valence" und "inscription", was redundant ist. Ich denke:
                                                                                                                                                            
  - Valence ist die Qualität der Relation: enables, constrains, constitutes, contradicts...                                                                 
  - Inscription wäre ein Name für die Relation selbst: "Die Förderung von X durch Y", oder auch nur "Förderung"
  - Oft fallen beide zusammen — die Valence ist die Inscription                                                                                             
                                                                                                                                                            
  Vorschlag für eine klarere Interaktion:                                                                                                                   
                                                                                                                                                            
  1. User wählt zwei Elemente                                                                                                                               
  2. UI fragt: Symmetrisch oder gerichtet?                  
    - Symmetrisch: A ↔ B (kein directed_from/to)                                                                                                            
    - Gerichtet: A → B (User wählt Richtung)                                                                                                                
  3. UI fragt: Wie benennen? (ein Feld statt zwei) — das wird sowohl Inscription als auch Valence                                                           
  4. Optional: eine Auswahl gängiger Valences als Vorschläge
